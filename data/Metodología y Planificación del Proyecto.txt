Metodología y Planiﬁcación de Proyecto 
 
1.  Metodología de Desarrollo 
 
Para el diseño, construcción e implementación del sistema se seleccionó una 
Metodología Ágil Iterativa e Incremental. Este enfoque es el más adecuado para 
proyectos que involucran Inteligencia Artiﬁcial y arquitecturas RAG (Retrival-
Augmented Generation), ya que permiten calibrar y reﬁnar los modelos de lenguaje, el 
tamaño de los fragmentos de información y las respuestas del agente basándose en 
pruebas continuas. 
 
El método se estructura en las siguientes etapas secuenciales iterativas, con 
productos especíﬁcos deﬁnidos para cada una: 
 
- Fase 1  Planiﬁcación, Requerimientos y Contexto:  En esta etapa inicial se 
deﬁne el alcance del sistema de atención y se recopilan los manuales técnicos, 
instructivos y reglamentos del área informática de la UFRO. Se establece el 
stack tecnológico base y se terminan las restricciones operativas.  
 
 Productos Asociados:  Documentos de problemática, alcance, objetivos, 
dominios del problema y estados del arte (tecnológico y cientíﬁco).  
 
- Fase 2  Especiﬁcación de Requerimientos, Diseño y Arquitectura:  
Corresponde a la deﬁnición formal de cómo operará el sistema. Aquí se modela 
la estructura estática y dinámica del sistema utilizando el modelo de vistas 4+1. 
Se deﬁnen las entidades de datos, los ﬂujos conversacionales, el pipeline de 
ingesta/búsqueda RAG y la topología física de despliegue.  
 
 Productos Asociados:  Documento de especiﬁcaciones de software (que 
incluye diagramas de casos de uso, de actividades y de estados), y el diseño 
de las arquitecturas estructural, funcional y “look & feel”.  
 
- Fase 3  Desarrollo Core (Backend/RAG) e Implementación del Frontend: 
Se codiﬁcan los endpoints en FastAPI y se conﬁgura LlamaIndex para el 
procesamiento O Ưline de documentos (lectura, fragmentación con 
TokenTextSplitter y generación de embbedings). Estos vectores se idexan 
localmente en la base de datos ChromaDB. Se integran las llamadas API hacia 
Claude 3.7 Sonnet para la lógica de orquestación y generación de respuestas 
enriquecidas con citas de fuentes universitarias oﬁciales. Paralelamente, se 
diseña e implementa la interfaz visual (Look & feel) del widget web en HTML/JS 
o React. Se realiza la integración con la Web Speech API nativa del navegador 
para habilitar de forma directa y sin costes los canales de entrada por vox 
(micrófono) y salida (síntesis de audio).  
 
 Productos Asociados:  Prototipo (MVP) con backend operativo, base de 
datos vectorial poblada y Widget Web integrado.  
 
- Fase 4  Pruebas, Validación y Cierre:  Se ejecutan pruebas de estrés con 
usuarios reales, evaluación de la calidad de las respuestas (evitando 
alucinaciones mediante el nodo Self-RAG) y pruebas de usabilidad. Se valida el 
correo funcionamiento del registro de logs en tiempo real hacia la API de Google 
Sheets y la monitorización avanzada en Langfuse. Se concluye con la entrega 
ﬁnal del prototipo, el empaquetado del widget mediante un script embebido y 
su integración ﬁnal dentro del Campus Virtual de la universidad.  
 
 Productos Asociados:  Informe ﬁnal del proyecto, sistema auditado, 
entrega ﬁnal y video.

2. Estrategia de Implementación 
 
La implementación del sistema consolida todas las entregas previas y futuras en un 
ciclo de desarrollo lógico. Comienza con la base teórica (problemática, objetivos y 
estados del arte), para luego traducir esa necesidad en requerimientos formales 
mediante el documento de especiﬁcaciones (con sus diagramas de casos de uso, 
actividades y estados). Posteriormente, esto se materializa en el diseño de las tres 
arquitecturas clave (estructuras, funcional y “look & feel”), lo cual permite construir y 
describir el Producto Mínimo Viable (MVP). El proyecto culmina con el informe ﬁnal y la 
presentación en video, evidenciando el funcionamiento de la solución implementada. 
 
Recursos a Utilizar: 
- Recursos Humanos:  Propietario del proyecto como desarrollador 
(considerando ingeniería de software, Arquitectura RAG y Diseño UI/UX).  
- Recursos Tecnológicos (Hardware):  Servidores locales para alojamiento del 
Bakcned (FastAPI) y la Base de Datos Vectorial (ChromaDB), y equipos de 
cómputo para desarrollo.  
- Recursos Tecnológicos (Software/Servicios):  Framework FastAPI, 
LlamaIndex, ChromaDB, Claude 3.7 Sonnet (Anthropic API), React/JS, Web 
Speech API, Google Sheets y Langfuse.  
 
Insumos: 
- Manuales técnicos, instructivos y reglamentos no estructurados del área 
informática y de soporte de la UFRO. 
- Documentación institucional requerida para alinear el “look & feel” a la 
identidad gráﬁca de la universidad. 
 
Resultados Evaluables y Medibles: 
1. Métrica de Relevancia y Precisión (RAG):  Porcentaje de respuestas generadas 
que incluyen citas exactas a los manuales UFRO sin presentar alucinaciones o 
información inventada.  
2. Métrica de Latencia:  Tiempo de respuesta del sistema desde que se recibe el 
audio/texto hasta que se sintetiza la respuesta en el navegador (objetivo: < 3 
segundos).  
3. Tasa de Resolución Autónoma:  Porcentaje de consultas de los estudiantes 
que son resueltas por el Widget si necesidad de escalar a la Mesa de Ayuda.  
4. Completitud de Entregables:  Cumplimiento del 100% de las entregas 
documentales requeridas (Arquitecturas, MVP, Informe Final, Video) en los 
plazos estipulados.  
 
 
3. Planiﬁcación, Seguimiento y Control 
 
Para garantizar el cumplimiento de los objetivos en los plazos estipulados, se 
estructuró una planiﬁcación temporal de 4 meses desde la selección de la 
problemática hasta el cierre del proyecto. 
 
Para el control de actividades, asignación de tareas y gestión del avance diario se 
utilizará Trello (o Github Projects), aplicando un tablero Kanban con columnas de 
estado: Por hacer (Backlog), En proceso (QA) y Hecho (Done). Esta herramienta ágil 
permitirá identiﬁcar cualquier tipo de problema de manera temprana durante las 
iteraciones de desarrollo del pepiline RAG.
